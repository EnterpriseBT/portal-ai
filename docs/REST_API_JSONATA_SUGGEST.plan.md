# REST API JSONata transform — AI suggestion — Plan

**TDD-sequenced implementation of the JSONata-transform suggestion feature: new contracts + error code, server-side `JsonataSuggester` dep + Haiku-backed default impl + prompt builder + array-slice truncation, a new `POST /api/connector-instances/suggest-transform` route with route-level validate-then-retry orchestration, an SDK mutation, a pure `TransformSuggesterUI`, and the container wiring in `ApiEndpointForm` that lights it up.** After this PR the JSONata transform editor has a Suggest button next to an optional prompt textarea. Clicking it (after Preview has captured a sample response) replaces the textarea contents with a JSONata expression generated by Haiku 4.5, server-side-validated against the captured sample.

Discovery: [`docs/REST_API_JSONATA_SUGGEST.discovery.md`](./REST_API_JSONATA_SUGGEST.discovery.md). Spec: [`docs/REST_API_JSONATA_SUGGEST.spec.md`](./REST_API_JSONATA_SUGGEST.spec.md). Standing on top of [`REST_API_PRE_COMMIT_PROBE`](./REST_API_PRE_COMMIT_PROBE.discovery.md), which shipped the editor, the preview pane, and the AI-assist precedent pattern (the column classifier).

Seven slices, each behind a green test suite. Slices 1–5 are server-only and add no user-visible change. Slice 6 adds the SDK endpoint (still no user-visible change). Slice 7 lights up the new UI in `ApiEndpointForm` — the only user-visible commit.

Run tests with `npm run test:unit` (per repo convention — `npx jest` bypasses the `NODE_OPTIONS=--experimental-vm-modules` flag the ESM-deps need). Type-check with `npm run type-check`.

Sequence rationale:

- **Slice 1** lands the shared contracts (`SuggestTransformRequestBodySchema`, `SuggestTransformResponseSchema`) + the new error code. Pure additions; no behavior change.
- **Slice 2** lands the `JsonataSuggester` dep contract + `JsonataSuggestError` class. Pure leaf, no caller.
- **Slice 3** lands the truncation util + the prompt builder. Pure leaf, no caller.
- **Slice 4** lands the Haiku-backed default suggester. Pure leaf consumed only by tests at this point.
- **Slice 5** lands the new route + the validate-then-retry orchestration. First server-observable change (the endpoint is reachable, no consumer yet).
- **Slice 6** lands the SDK endpoint. Wires the mutation hook; no caller in the workflow yet.
- **Slice 7** lands `TransformSuggesterUI` and the container wiring in `ApiEndpointForm`. First user-visible change.

After every slice the repo type-checks, `npm run test:unit` is green, and (through slice 6) the workflow's user-visible behavior is unchanged.

## Slice 3 — `truncateForPrompt` + `buildJsonataSuggestPrompt`

Two pure functions in one file. Both side-effect-free. Together they're the deterministic prompt-construction layer.

**Files**

- New: `apps/api/src/adapters/rest-api/jsonata-suggest.prompt.ts`
- New: `apps/api/src/__tests__/adapters/rest-api/jsonata-suggest.prompt.test.ts`

**Contract**

```ts
export function truncateForPrompt(value: unknown): unknown;

export interface BuildJsonataSuggestPromptInput {
  sampleResponse: unknown;        // already truncated
  promptHint?: string;
  previousAttempt?: { expression: string; error: string };
}

export function buildJsonataSuggestPrompt(
  input: BuildJsonataSuggestPromptInput,
): string;
```

**Steps**

1. **Write truncation tests:**
   1. Primitives pass through unchanged: `truncateForPrompt(42)` returns `42`, `null` returns `null`, `"abc"` returns `"abc"`, `false` returns `false`, `undefined` returns `undefined`.
   2. Array of ≤5 elements returns a new array with each element recursed (deep-equal but not reference-equal).
   3. Array of 6 elements returns a 6-element array: first 5 elements (recursed) + `"__truncated__"` as the 6th.
   4. Array of 100 elements returns a 6-element array (first 5 + sentinel).
   5. Nested array — `[[1, 2, 3, 4, 5, 6], [7]]` returns `[[1, 2, 3, 4, 5, "__truncated__"], [7]]`. Outer array passes through (length 2); inner gets sliced.
   6. Plain object passes through with values recursed: `{ a: [1, 2, 3, 4, 5, 6] }` becomes `{ a: [1, 2, 3, 4, 5, "__truncated__"] }`.
   7. Deeply nested: `{ data: { items: [/* 10 objects */] } }` — items array slice + sentinel; each surviving object recursed.
   8. Mixed types in array of length >5 — first 5 elements preserved verbatim regardless of type, sentinel appended.
   9. Empty array returns empty array (not the sentinel).
   10. Empty object returns empty object.
   Run; all fail.

2. **Author `truncateForPrompt`.** Recursive switch on `value === null` / `Array.isArray(value)` / `typeof value === "object"` / else. Slice arrays at 5, append the literal sentinel string. Recurse into elements before slicing (so deep arrays inside large outer arrays still get processed for the elements that survive).

3. **Write prompt-builder tests:**
   1. With just `sampleResponse`, no hint, no previous attempt — output contains `## Sample response\n{...JSON.stringify(sample, null, 2)...}`, contains `(no hint provided)` under `## User hint`, and does **not** contain the `## Previous attempt` heading.
   2. With `promptHint: "use just id and email"` — `## User hint` section renders the hint string verbatim.
   3. With `previousAttempt: { expression: "data.items.{}", error: "the expression returned 0 records" }` — `## Previous attempt` section is present and includes both fields.
   4. Without `previousAttempt`, the `## Previous attempt` heading is absent (asserts no orphaned empty section).
   5. The closing instruction line `Return JSON: { "expression": "<jsonata expression string>" }.` is present once.
   6. The prompt is deterministic: two calls with deep-equal input produce strictly-equal strings.
   7. The opening rubric lines (the four bullet points about "must return an array", "no JavaScript", etc.) are present in order.
   8. JSON.stringify uses `null, 2` indentation (assert by checking for `\n  ` after the `## Sample response` header).
   Run; all fail.

4. **Author `buildJsonataSuggestPrompt`.** Compose strings from the input. Lifts the formatting style from `buildApiClassifierPrompt` (lines 56–90 of `classifier.prompt.ts`).

5. **Run tests.** All cases green.

**Done when:** truncation handles every shape; prompt builder is deterministic and conditionally renders sections.

```ts
// Inside connector-instance.router.ts or a small helper file beside it.
function isStrictlyValidRecords(result: TransformResult): boolean {
  if (result.error !== null) return false;
  if (result.records.length === 0) return false;
  return result.records.every(
    (r) => typeof r === "object" && r !== null && !Array.isArray(r),
  );
}

function formatValidationError(result: TransformResult): string {
  if (result.error) return `applyTransform reported a ${result.error.kind} error: ${result.error.message}`;
  if (result.records.length === 0)
    return "the expression returned 0 records; the response shape needs an expression that produces at least one record";
  return "the expression returned non-object records (got primitives or nested arrays); each record must be a plain object";
}
```

2. **Add the route**. Place it next to `probe-endpoint-draft` and `preview-endpoint-page` (lines 826–end). Structure:
   ```ts
   connectorInstanceRouter.post(
     "/suggest-transform",
     getApplicationMetadata,
     async (req, res, next) => {
       try {
         const parsed = SuggestTransformRequestBodySchema.safeParse(req.body ?? {});
         if (!parsed.success) {
           throw new ApiError(400, ApiCode.REST_API_INVALID_CONFIG, ...);
         }
         const { promptHint, sampleResponse } = parsed.data;
         const truncated = truncateForPrompt(sampleResponse);
         const suggester = getJsonataSuggester();

         const first = await suggester.suggest({ sampleResponse: truncated, promptHint });
         const v1 = await applyTransform(first.expression, sampleResponse);
         if (isStrictlyValidRecords(v1)) {
           return HttpService.success(res, { expression: first.expression, warning: null }, 200);
         }

         const second = await suggester.suggest({
           sampleResponse: truncated,
           promptHint,
           previousAttempt: { expression: first.expression, error: formatValidationError(v1) },
         });
         const v2 = await applyTransform(second.expression, sampleResponse);
         if (isStrictlyValidRecords(v2)) {
           return HttpService.success(res, { expression: second.expression, warning: null }, 200);
         }
         return HttpService.success(
           res,
           { expression: second.expression, warning: { kind: "validation-failed", message: formatValidationError(v2) } },
           200,
         );
       } catch (error) {
         logger.error(...);
         if (error instanceof JsonataSuggestError) {
           return next(new ApiError(502, ApiCode.REST_API_TRANSFORM_SUGGEST_FAILED, error.message, { reason: error.reason }));
         }
         return next(error instanceof ApiError ? error : new ApiError(500, ApiCode.REST_API_OPERATION_FAILED, ...));
       }
     },
   );
   ```

## Risk + rollback notes

- **Worst-case failure mode:** the Haiku model returns garbage on every call, and the validation retry doesn't catch a class of broken expressions (e.g. an expression that returns an array of `null`s — `error: null`, `records.length > 0`, every record fails the `typeof === "object" && r !== null` check). The strict validation in the route filters that case (one of the `r === null` records fails the strict check), so the warning path triggers. Worst-worst case: the user sees a warning + a useless expression, then hand-edits it or clicks Suggest again.
- **No data persistence in this PR.** Rollback is `git revert <commit-range>` + `npm run build` — no DB rollback needed.
- **Anthropic provider scope.** The same Haiku model is already used by the classifier; this PR adds no new external dependency, just a new prompt against the existing provider. No new secrets, no new env vars.
- **Cost containment.** Two model calls per Suggest click in the worst case. The button is disabled until Preview captures a sample, so the user can't spam it for nothing. If telemetry shows abuse, layer caching (per the spec's deferred follow-ups).
